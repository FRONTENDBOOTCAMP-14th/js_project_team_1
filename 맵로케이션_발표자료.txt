맵로케이션 기능 발표 자료
==============================================

📍 1. 각 영역(HTML / CSS / JS) 키 포인트

🔹 HTML 구조
- 좌측: 장소 정보 카드 + 날씨 정보 표시 영역
- 우측: 카카오 지도 컨테이너 + 주소 표시바
- 시맨틱 태그 활용 (aside, main, section)
- ID와 클래스로 JavaScript 조작 가능하게 설계

🔹 CSS 스타일링
- CSS 변수(Custom Properties) 활용한 일관된 디자인 시스템
- Flexbox 레이아웃으로 반응형 구현
- 카드형 UI 디자인 (그림자, 둥근 모서리)
- Pretendard 폰트로 깔끔한 타이포그래피

🔹 JavaScript 기능
- ES6 클래스로 코드 구조화 (MapLocationApp)
- 카카오 지도 API 동적 로딩
- 비동기 처리 (async/await) 활용
- 이벤트 기반 상호작용 구현

📍 2. 어려웠던 점 & 해결 과정

🔹 주요 문제점과 해결 과정

1️⃣ 카카오 지도 API 로딩 문제
- 문제: 처음에 지도가 제대로 안 뜨는 현상
- 해결: loadKakaoMapsAPI() 함수로 동적 로딩 구현
- 배운점: 외부 API는 Promise로 로딩 상태 관리해야 함

2️⃣ 좌표-주소 변환 (지오코딩)
- 문제: 클릭한 지점의 정확한 주소 얻기
- 해결: 카카오 Geocoder 서비스 활용
- 추가: 근처 관광지 이름도 함께 검색해서 표시

3️⃣ 날씨 API 연동
- 문제: CORS 오류와 API 키 관리
- 해결: 환경변수(.env)로 API 키 분리
- 추가: OpenWeatherMap API로 실시간 날씨 정보

4️⃣ 클립보드 복사 기능
- 문제: 브라우저별 호환성 이슈
- 해결: navigator.clipboard + 폴백 방식 병행
- 배운점: 보안 정책 때문에 HTTPS에서만 작동

📍 3. 데모 후 받은 피드백과 이를 반영한 사례 소개

🔹 데모 시연 및 결과 공유

💡 기능 시연
1. 지도에서 원하는 위치 클릭
2. 해당 위치의 실시간 날씨 정보 표시
3. 주소 복사 기능으로 편리한 공유
4. URL 파라미터로 특정 장소 바로 이동

💡 받은 피드백과 개선사항
1. "지도 크기를 좀 더 크게 해주세요"
   → CSS에서 지도 영역 비율 조정

2. "날씨 정보가 더 자세했으면 좋겠어요"
   → 체감온도, 습도, 풍속까지 추가 표시

3. "모바일에서도 써보고 싶어요"
   → 반응형 CSS로 모바일 최적화 진행

4. "로딩 시간이 좀 걸려요"
   → setTimeout으로 지도 초기화 딜레이 최적화

💡 실제 활용 사례
- 여행 계획 수립 시 목적지 날씨 확인
- 야외 활동 전 현장 날씨 체크
- 친구들과 만날 장소의 날씨 공유

💡 앞으로 추가하고 싶은 기능
- 시간대별 날씨 예보
- 즐겨찾기 장소 저장
- 소셜 공유 기능
